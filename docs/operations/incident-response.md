# Cloud and AI incident procedure

## Triage

Record incident ID, evaluation/correlation IDs, detection source, time, affected boundary, synthetic/real-data determination, and current service state. Classify severity without relying on AI-generated conclusions.

## Contain

- Disable the consequential tool or model adapter if authorization or behavior is uncertain.
- Remove an adversarial corpus item from the active manifest while preserving controlled evidence.
- Scale the SUT to zero if cost or abuse is growing.
- Remove or isolate an unsafe fixture; never attach a broad NSG to compute.
- Revoke a compromised federated credential or role assignment using the approved bootstrap path.

## Preserve and investigate

Preserve raw private evidence, sanitized derivative, configuration and model digests, Azure Activity/KQL results, decision events, and manifest signatures. Record failed collections as limitations. Do not copy raw prompt/response content into routine tickets or logs.

## Recover and learn

Redeploy from an approved immutable digest, run the complete affected evaluation set, perform a clean assessment, and require fresh reviewer decisions. Update threat model, cases, controls, and procedures. Finding closure requires new retest evidence; service recovery alone is insufficient.
